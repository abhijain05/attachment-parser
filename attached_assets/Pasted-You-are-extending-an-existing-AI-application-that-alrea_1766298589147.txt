You are extending an existing AI application that already supports
OpenAI and Gemini for embeddings.

Your task is to ADD ONE MORE EMBEDDING PROVIDER:
"Self-Hosted VPS (Ollama)"

This VPS hosts Ollama models and exposes a secure HTTP API via a gateway.

────────────────────────────────
EMBEDDING PROVIDERS TO SUPPORT
────────────────────────────────

1) OpenAI
2) Gemini
3) VPS (Ollama)  ← NEW

The application must allow the user to choose the embedding provider at runtime.

────────────────────────────────
VPS EMBEDDING DETAILS (IMPORTANT)
────────────────────────────────

• Provider name: "vps"
• Embedding model (fixed): nomic-embed-text
• Gateway URL: http://31.97.210.209:8001/embeddings
• Authentication: API key via HTTP header

Header:
  api-key: <USER_CONFIGURED_API_KEY>

Request (JSON):
{
  "input": "Text to embed"
}

Response (JSON):
{
  "embedding": [float, float, ...]
}

This endpoint returns a single vector array.
No batching is required initially.

────────────────────────────────
IMPLEMENTATION REQUIREMENTS
────────────────────────────────

1. Create a unified embedding interface:
   getEmbedding(provider, text)

2. Routing logic:
   - if provider === "openai" → existing OpenAI logic
   - if provider === "gemini" → existing Gemini logic
   - if provider === "vps" → call VPS gateway

3. The VPS embedding call MUST:
   - Use fetch / axios
   - Pass api-key header
   - Handle network errors
   - Validate embedding length > 0

4. The returned embedding must be normalized into a float array
   and stored exactly like OpenAI/Gemini embeddings.

────────────────────────────────
EXAMPLE USAGE (APPLICATION SIDE)
────────────────────────────────

User selects:
  Embedding Provider: VPS (Self-Hosted)

Code example:

const embedding = await getEmbedding(
  "vps",
  "Senior SAP UI5 developer with 10 years experience"
)

storeEmbedding(embedding)

────────────────────────────────
ERROR HANDLING
────────────────────────────────

If VPS call fails:
- Return a structured error:
  "VPS embedding service unavailable"
- Do NOT crash the app
- Allow fallback to OpenAI or Gemini

────────────────────────────────
SECURITY RULES
────────────────────────────────

• VPS URL must be configurable via environment variable
• API key must NEVER be hardcoded
• API key must be sent only via headers

────────────────────────────────
GOAL
────────────────────────────────

After implementation:
- The app supports OpenAI, Gemini, and VPS embeddings
- The user can switch providers without changing business logic
- VPS embeddings are stored and used identically to other providers

Proceed with implementation.
